# Agent 逻辑清单与 MultiAgent-Stock.docx 核对

## 一、所有 Agent 简要逻辑清单

### 主控层（1个）

| Agent | dim_code | 简要逻辑 |
|-------|----------|----------|
| Master Orchestrator | MASTER | 统一调度、一致性检测、辩论仲裁、加权评分；五级决策（强烈卖出/弱卖出/观望/弱买入/强烈买入）；短线/中长线持有建议（基于周线、月线动量） |

### 核心层（9个，必选）

| Agent | dim_code | 简要逻辑 |
|-------|----------|----------|
| 趋势与结构分析师 | TREND | 50 + 0.45×动量 + 0.5×趋势强度 + 0.15×涨跌幅 + 0.25×回撤；多周期K线（仅日线/周线/月线）、HH/HL/LH/LL结构 |
| 技术指标量化分析师 | TECH | 50 + 0.35×动量 + 0.3×趋势 + (量比-1)×8 - 0.2×波动；RSI/MACD/KDJ/布林带/均线 |
| 流动性与盘口分析师 | LIQ | 50 + (量比-1)×10 - 0.2×|涨跌幅| - 0.1×波动 + 换手偏置；盘口、成交量、换手率 |
| 资金流向分析师 | CAPITAL_FLOW | 50 + 0.25×涨跌幅 + (量比-1)×10 + 0.08×新闻情绪 + 0.15×动量；北向、主力、龙虎榜、股东户数 |
| 大盘联动分析师 | BETA | 50 + 0.2×趋势 + 0.12×涨跌幅 - 0.25×波动；与沪深300相关性、Beta |
| 板块与政策分析师 | SECTOR_POLICY | 50 + 0.07×新闻情绪 + 0.1×动量；申万行业、板块轮动、产业政策 |
| 舆情分析师 | SENTIMENT | 50 + 0.45×新闻情绪 + 0.1×涨跌幅；新闻、公告、政策情绪 |
| 基本面分析师 | FUNDAMENTAL | 50 + PE偏置 + PB偏置 + 0.1×动量；PE/PB、经营质量 |
| 量化策略分析师 | QUANT | 50 + 0.3×动量 - 0.15×波动 + (量比-1)×6；动量、波动、量比信号 |

### 专家层（5个，可选）

| Agent | dim_code | 简要逻辑 |
|-------|----------|----------|
| 宏观关联分析师 | MACRO | 50 + 0.1×动量 - 0.1×波动 + 0.05×新闻情绪；宏观市场关联 |
| 产业链分析师 | INDUSTRY | 50 + 0.08×新闻情绪 + 0.12×动量；行业产业链健康 |
| 精细资金流分析师 | FLOW_DETAIL | 50 + (量比-1)×12 + 0.15×涨跌幅；超大单/大单/中单/小单流向 |
| 主力行为分析师 | MM_BEHAVIOR | 50 + (量比-1)×8 + 0.2×新闻情绪 - 0.15×波动；主力操盘手法 |
| 深度舆情分析师 | NLP_SENTIMENT | 50 + 0.55×新闻情绪 + 0.1×动量；NLP情绪网络 |

### 结构研判层（2个）

| Agent | dim_code | 简要逻辑 |
|-------|----------|----------|
| 顶部结构分析师 | TOP_STRUCTURE | 基于日线/周线/月线K线：长上影线+负动量、高振幅+负动量（头肩顶/M顶/长上影线）→ 减分；长下影线+正动量 → 加分；评分=50 - top_signals + bot_signals×0.3 |
| 底部结构分析师 | BOTTOM_STRUCTURE | 基于日线/周线/月线K线：长下影线+正动量、正动量反弹（W底/长下影线阳线）→ 加分；长上影线+负动量 → 减分；评分=50 + bot_signals - top_signals×0.3 |

### 报告增强层（6个，信息型不参与评分）

| Agent | dim_code | 说明 |
|-------|----------|------|
| 公司概览分析师 | COMPANY_OVERVIEW | 公司基本信息、行业地位、结构化概览表 |
| 关键价位Fibonacci分析师 | KEY_LEVELS | 关键价位、Fibonacci回撤位 |
| 竞争格局分析师 | COMPETITION | CR3/CR5、竞争对手矩阵、核心优势 |
| 综合风险管理分析师 | RISK_MANAGEMENT | 多类型风险等级与详细说明 |
| 情景分析师 | SCENARIO | 乐观/中性/悲观情景、概率、触发条件、价格目标 |
| 仓位与止损策略分析师 | POSITION_STRATEGY | 分批建仓、止损位、介入条件、催化剂 |

## 三、差异与改进建议

### 3.1 实现方式差异

| 项目 | docx 描述 | 当前实现 |
|------|-----------|----------|
| 数据获取 | 模式A：Tool自取 / 模式B：预取注入 | 模式B：Tushare/AKShare 预取，Agent 仅分析 |
| 评分公式 | 每Agent输出 s_i ∈ [0,100]，加权 S = Σ w_i·s_i | ✓ 一致 |
| 决策映射 | S≥70 强势, 50≤S<70 中性, S<50 弱势 | 扩展为五级：强烈买入(≥85)、弱买入(70-85)、观望(50-70)、弱卖出(40-50)、强烈卖出(<40) |
| 短线/中长线 | docx 未明确 | 已增加：短线建议、中长线建议（基于周线、月线动量） |

### 3.2 改进建议

1. **TOP_STRUCTURE / BOTTOM_STRUCTURE 形态识别**
   - 当前：基于上/下影线比例、动量、振幅的量化规则
   - 建议：可考虑引入更精细的形态识别（如头肩顶需3峰结构、M顶需双顶确认），或接入 LLM 对 K 线形态做语义描述增强

2. **月线数据完整性**
   - 当前：月线已纳入抓取与 kline_indicators，但 `required_tfs` 不含 month，月线失败不阻塞 is_complete
   - 建议：若中长线建议依赖月线，可考虑在月线缺失时明确标注「月线数据不足」

3. **权重与 docx 对齐**
   - docx 核心9维有默认权重（如 TREND 0.18、TECH 0.12 等），当前实现通过 configs/agents/*.json 配置
   - 建议：核对各 agent 的 weight 与 docx 默认权重是否一致

4. **Agent 产出格式**
   - docx 要求：要点式结构总结、关键价位、缺口清单、多级别传导路径等
   - 当前：以评分+reason 为主，LLM 可丰富 reason
   - 建议：若需严格对齐 docx，可在 Agent 输出中增加结构化字段（如 key_levels、gaps、path 等）

5. **数据必须项**
   - docx：日K线≥60根、周K线≥30根；技术指标需日K线≥120根
   - 当前：timeframes 中 day/week 各 100 根，month 60 根
   - 建议：确认各数据源实际返回根数满足 docx 最低要求

## 四、总结

- **架构**：主控1 + 核心9 + 专家5 + 结构研判2 + 报告增强6 = 23 个 Agent；已删除 DERIV_MARGIN、ARB、SHAREHOLDER、QUALITY、REGULATION 共5个，QUANT 移入核心。
- **逻辑**：各 Agent 的 dim_code、角色、数据依赖与 docx 一致；评分公式为基于特征的规则计算，可被 LLM 增强。
- **报告**：已实现五级决策、短线/中长线建议、各 Agent 简要研判分析、评分汇总表。
- **改进方向**：形态识别精细化、月线完整性提示、权重与产出格式与 docx 进一步对齐。
