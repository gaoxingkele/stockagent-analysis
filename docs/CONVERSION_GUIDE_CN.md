# 他类智能体设计文档 → 中国股市智能体定义 文档转换规范

当你提供“其他品类（如商品、外汇、加密、海外股票）智能体设计文档”后，按以下规则转换：

## 1. 转换目标

- 保留原文档中的角色协作思想与流程框架
- 将业务语义替换为中国股市语义
- 产出可直接用于本项目的智能体定义文档与配置草案

## 2. 关键映射规则

- 标的对象 → 中国 A 股个股（`symbol`）
- 交易结论 → `buy/hold/sell`
- 数据接口 → `AKShare/Tushare`（或在备注里说明未支持字段）
- 风险模块 → 政策风险、流动性风险、回撤风险、风格切换风险

## 3. 必须保留的输出结构

1. 角色定义（manager + 分析智能体）
2. 交互协议（消息 JSON 格式）
3. 投票机制（权重、阈值）
4. 数据源说明（全局+智能体专属）
5. 日志与审计字段（可追溯）

## 4. 工具脚本

使用命令：

```bash
python run.py convert-doc --input "D:\path\your-source.md"
```

输出目录：

- `docs/converted/*_cn_stock.md`

## 5. 人工复核建议

- 复核术语是否已完全替换为中国股市语境
- 复核数据字段是否能在 AKShare/Tushare 中取得
- 复核角色职责是否与本项目的 manager 编排机制一致
