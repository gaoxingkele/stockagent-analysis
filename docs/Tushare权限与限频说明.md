# Tushare 权限与限频说明

参考：[积分与权限](https://tushare.pro/document/1?doc_id=108)、[积分频次表](https://tushare.pro/document/1?doc_id=290)

## 积分与频次（常规数据）

| 积分数 | 每分钟频次 | 每天总量 |
|--------|------------|----------|
| 120 | 50 | 8000次 |
| 2000+ | 200 | 100000次/API |
| **5000+** | **500** | 常规数据无上限 |
| 10000+ | 500 | 常规+特色数据 |

## 分钟数据需单独开通

**股票历史分钟**（1/5/15/30/60 分钟）需单独开通权限，**不在积分范畴**：

- 开通方式：捐助 2000 元/年
- 频次：500 次/分钟

若未开通，调用 `pro_bar` 分钟数据可能返回「每分钟最多访问该接口 2 次」。

## 本项目的应对

1. **调用节流**：每次 Tushare 调用前间隔 5 秒（可配置 `TUSHARE_CALL_INTERVAL_SEC`）
2. **连接超时**：默认 15 秒（可配置 `TUSHARE_TIMEOUT_SEC`），超时后立即改用 AKShare
3. **权限/限频**：收到「每天最多」「每分钟最多」「限频」等错误时，立即改用 AKShare（分钟数据需单独开通权限）
4. **多源备选**：Tushare 失败后自动改用 AKShare

## 环境变量

```bash
# 调用间隔，默认 5 秒
TUSHARE_CALL_INTERVAL_SEC=5

# 连接超时，默认 15 秒（超时后立即改用 AKShare）
TUSHARE_TIMEOUT_SEC=15

# 或关闭节流（0 表示不等待）
TUSHARE_CALL_INTERVAL_SEC=0
```

## 5000 积分用户建议

- 日线/周线：500 次/分钟，通常无需额外配置
- 分钟线（30m/1h/2h）：需开通「股票历史分钟」权限，否则会限频
